Data Flow Diagrams (DFD) Analysis Report

Project: MediQ Antibiotic Management App | DFD Standards: Yourdon-DeMarco & Gane-Sarson System

1. Data Flow Diagram Reference Guide

This report covers the logical DFD for the MediQ Antibiotic Management System, decomposing it into processes, entities, data stores, and data flows mapped directly to the actual Firestore database structure.

Symbol Type Standard Representation Application in MediQ
Process Rounded Rectangle / Bubble Transforms data inputs (e.g., login checking, recording usages, compiling usage stats).
External Entity Flat Rectangle Source or destination of data outside system boundary (e.g., Admin, Pharmacist, Cloudinary, Firebase Auth).
Data Store Open Rectangle (Double/Single open end) Represents Firestore Database Collections (e.g., users, wards, antibiotics, book_numbers, main_stock, releases, return_stock, returns).
Data Flow Arrow Line Directs paths for data movement with descriptive label noun-phrase.

3. DFD Level 1 - Detailed Functional Processes

Level 1 expands the system's boundary into 8 key processes and maps them directly to the Firestore collection-based data stores.

  • 1.0 Authenticate & Route User: Validates user details against Firebase Auth, checks user profiles from D1: users, and routes them to the proper Dashboard.
  • 2.0 Manage User Profiles: Admin utility to add, edit, or remove user data from D1: users.
  • 3.0 Manage Wards & Antibiotics: Admin configuration panel which reads/writes information in D2: wards and D3: antibiotics.
  • 4.0 Manage Book Registers: Logs registered book range updates to D4: book_numbers.
  • 5.0 Manage Stock Inventory: Tracks main stock updates in D5: main_stock.
  • 6.0 Record Releases & Returns: Logs pharmacist actions to D6: releases, D7: return_stock, and D8: returns.
  • 7.0 Compile Usage Statistics: Extracts logs from D6: releases and D8: returns to compile charts.
  • 8.0 Edit Profile Settings: Posts image updates to Cloudinary and stores data in D1: users.
