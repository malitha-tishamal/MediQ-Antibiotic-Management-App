Why MediQ?

In Sri Lankan government hospitals, pharmacists manually manage antibiotic releases and returns in logbooks. This is time-consuming, prone to calculation errors, and vulnerable to data loss. With the rising global threat of Antimicrobial Resistance (AMR), tracking antibiotic consumption categories (AWaRe) is critical for medical stewardship.

The Problem

Manual registers offer zero real-time inventory oversight, leading to sudden out-of-stock crises or drug expiration.

The Solution

An automated, cloud-synced platform that logs transaction histories, generates charts instantly, and keeps record logs in sync.

Parallel Stewardship

Keeps book and page number fields mandatory, allowing pharmacies to operate digitally while keeping physical registers as backups.

System Capabilities

Full-Suite Features

Role-Based Access

Distinct dashboards and workflows for Administrators (Inventory management, User approvals) and Pharmacists (Releases, Returns).

Failed Login Lockout

Increments failed login attempts and triggers progressive lockout timers, locking account permanently after 8 unsuccessful tries.

Interactive Analytics

Rich graphical pie & bar charts using Flutter's FL Chart package mapping antibiotic consumption across categories and wards.

PDF Certificate & Exports

Export all transactions to CSV for auditing or compile visual main store state summaries into structured PDF certificates.

Atomic Transactions

Utilizes Cloud Firestore transactional rules to ensure stock count decrements synchronously upon pharmacist releases.

Cloudinary Compression

Processes and compresses profile images locally prior to uploading and saving remote URL links to Firestore.
